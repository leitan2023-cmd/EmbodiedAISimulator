# EmbodiedAISimulator

Unity prototype for loading MANSION-generated multi-floor building JSON into an explorable scene for embodied AI research.

This repository currently focuses on:

- loading `floor_*.json` scene descriptions from MANSION / MansionWorld
- reconstructing room floors and wall geometry in Unity
- loading available Objathor assets
- falling back to semantic proxy furniture when full assets are unavailable
- floor switching and simple first-person exploration

It is a practical scene viewer / simulator scaffold, not yet a complete embodied training stack.

## Repository Structure

- `Assets/Scripts/Builder/MansionSceneBuilder.cs`
  - Main runtime loader for floors, walls, objects, and runtime stats.
- `Assets/Scripts/Data/FloorData.cs`
  - JSON data model for the MANSION floor format used here.
- `Assets/Scripts/Data/ObjathorMeshData.cs`
  - Data model for converted Objathor mesh payloads.
- `Assets/Scripts/Util/PolygonTriangulator.cs`
  - Room floor triangulation via `LibTessDotNet`.
- `Assets/Scripts/FloorManager.cs`
  - Controls active floor visibility.
- `Assets/Scripts/FloorHotkeys.cs`
  - Numeric floor switching hotkeys.
- `Assets/Scripts/SimpleFPSCamera.cs`
  - Minimal WASD + mouse look + `Q/E` vertical motion.
- `Assets/StreamingAssets/Buildings/test_building`
  - Example building JSON copied from MansionWorld.

## Prerequisites

Recommended environment:

- Unity 6 / Unity 2023+ compatible editor
- URP project template
- macOS, Windows, or Linux editor

- `LibTessDotNet` source under `Assets/Plugins/LibTessDotNet`

## Asset Layout

This project expects the following runtime layout:

- `Assets/StreamingAssets/Buildings/<building_name>/floor_1.json`
- `Assets/StreamingAssets/Buildings/<building_name>/floor_2.json`
- `Assets/StreamingAssets/ObjathorAssets`

In the current local setup, `ObjathorAssets` is intended to be a symlink to the downloaded Objathor asset folder:

```bash
ln -s ~/.objathor-assets/2023_09_23/assets \
  /path/to/UnityProject/Assets/StreamingAssets/ObjathorAssets
```

The symlink itself is intentionally ignored by git, because the asset payload is large and machine-local.

## Open And Run

1. Open this repository as a Unity project.
2. Open `Assets/Scenes/SampleScene.scene`.
3. In the scene, create or keep an empty GameObject named `MansionLoader`.
4. Attach these components to `MansionLoader`:
   - `MansionSceneBuilder`
   - `FloorManager`
   - `FloorHotkeys`
5. Attach `SimpleFPSCamera` to `Main Camera`.

Recommended first-run settings on `MansionSceneBuilder`:

- `Building Folder`: `test_building`
- `Initial Floor`: `1`
- `Floor Height`: `3.0`
- `Build Nav Mesh`: enabled
- `Use Semantic Proxy Fallback`: enabled
- `Skip Doorway Placeholders`: enabled
- `Log Furniture Stats`: enabled

`Wall Mat` and `Floor Mat` can be left empty. The loader creates basic URP/Lit fallback materials at runtime.

## Controls

Scene navigation:

- `W A S D`: move
- mouse: look around
- `Q`: move downward
- `E`: move upward
- `Esc`: release cursor

Floor control:

- `1` to `9`: switch visible floor
- numpad `1` to `9`: also supported

## What Happens On Play

When you press Play, the loader:

1. scans `Assets/StreamingAssets/Buildings/<building_name>` for `floor_*.json`
2. creates one root GameObject per floor
3. builds room floor meshes from room polygons
4. builds wall quads from wall polygons
5. tries to load object assets in this order:
   - `<assetId>.glb`
   - `<assetId>.mesh.json.gz`
   - semantic proxy furniture
   - final small placeholder
6. activates only the selected initial floor
7. optionally bakes one NavMesh per floor

Console output includes per-floor object loading statistics, for example:

```text
Floor 1: GLB=0, Converted=176, Proxy=40, Placeholder=4, Skipped=4
```

## Why Some Objects Are Real Meshes And Some Are Proxies

MANSION / MansionWorld scene JSON often references a mix of:

- hashed Objathor asset ids that exist locally
- symbolic names like `RoboTHOR_office_chair_volmar` or `Desk_320_2_1`

Only a subset of those names map directly to local, Unity-loadable assets in the current setup.

Current strategy:

- hashed assets with converted mesh payloads are rendered as real geometry
- unresolved named assets become semantic proxy furniture
- unresolved doors / doorway markers can be skipped to reduce clutter

This makes the scene readable and useful for prototyping even without a full asset remapping database.

## Recommended LLM Workflow For Embodied AI

The safest pattern is:

```text
LLM requirement
-> MANSION scene planning
-> JSON validation / rule checks
-> Unity loading
-> task generation
-> agent training / evaluation
```

Best practice:

- let the LLM plan building purpose, room composition, topology, and object selection
- do not let the LLM directly control runtime object transforms in Unity
- enforce training constraints with code, not prompt-only instructions

Useful training-time checks include:

- room existence and required room types
- door accessibility
- corridor clearance
- object overlap thresholds
- target object reachability
- multi-floor path feasibility

## Known Limitations

- wall openings for doors/windows are not yet cut into wall meshes
- many named RoboTHOR-style assets still use semantic proxies
- no full embodied agent controller is included yet
- no task sampler / reward system is included yet
- object placement is approximate and may still need per-category refinement
